# Paintflow Studio — GUI版マニュアル

paintflowエンジンに直感操作のGUIを被せたデスクトップアプリ。
リアルタイムプレビューを見ながらスライダーで絵作りし、領域をクリックで塗り替え、
タイムラインでアニメを付けてPNG連番/GIFに書き出せる。

技術スタックは **PySide6 (Qt)** — Houdini / Maya / Nuke のツールUIと同じQt系なので、
ここで書いたUIコードはそのままDCCツール開発に応用できる。

---

## 1. すぐ起動する(exe化しない場合)

```
pip install -r requirements.txt
```

を一度やってあれば、**`run_studio.bat` をダブルクリック**するだけで起動する。
(またはコマンドで `python studio_main.py`)

## 2. exe化する

**`build_exe.bat` をダブルクリック**。初回は5〜10分かかる。完了すると

```
dist\PaintflowStudio\PaintflowStudio.exe
```

ができる。以後は**このexeをダブルクリックするだけ**で起動(Python不要)。
`dist\PaintflowStudio` フォルダごと別PCにコピーしても動く。

> 確認済み: Python 3.14 は PySide6 6.11+ / PyInstaller 6.21(3.8〜3.15対応)でサポートされている。
> もし `pip install PySide6` が失敗する場合は `python -m pip install --upgrade pip` してから再実行。

## 3. 画面と操作

```
┌────────────┬──────────────────────────┬──────────┐
│ パラメータ  │        キャンバス          │ レイヤー  │
│ (左ドック)  │   ホイール: ズーム          │ 表示切替  │
│ 線画抽出    │   中ドラッグ/Space: パン    │ 完成      │
│ 自動彩色    │   左クリック: バケツ塗り     │ 歪み前    │
│ 線の合成    │   右クリック: 上書き解除     │ フラット  │
│ 水滴ひずみ  │   ダブルクリック: フィット   │ 線画 ほか │
├────────────┴──────────────────────────┴──────────┤
│ タイムライン  ▶再生 | 長さ/fps | トラック追加 | ◆+キー      │
└──────────────────────────────────────────────────┘
```

### 基本フロー
1. 画像を**ウィンドウにドラッグ&ドロップ**(または ファイル→開く)
2. 左のスライダーを動かすと**リアルタイムでプレビュー更新**
   (melt等のひずみ系は特に軽い — ステージ単位キャッシュで差分だけ再計算している)
3. 色が気に入らなければ「シード🎲」を回すか、**バケツツール**へ:
   - ツールバーの「バケツ色」で色を選び、**キャンバスの領域を左クリック**→その領域だけ塗り替え
   - **右クリック**でその領域の上書きを解除、「上書きクリア」で全解除
   - 上書きは正規化座標で保持されるので、**フル解像度書き出しでも同じ領域に効く**
4. 🖼 PNG書き出し(フル解像度)

### アニメを付ける
1. タイムラインでトラックを選んで「トラック追加」(まずは「にじみ進行」= drip.melt)
2. 再生ヘッドを 0s に置き、meltスライダーを 0 にして「◆ +キー」
3. 再生ヘッドを 1.5s あたりに動かし、meltを 1.0 にして「◆ +キー」
4. ▶再生でループ確認。◆をクリックすると値/イージングを下の欄で編集できる
5. 🎞 アニメ書き出し → フォルダ指定 → PNG連番 + preview.gif が出る

### プリセット
ファイル→プリセット保存/読込。パラメータ+パレット+色上書き+タイムラインが
まるごとJSONに入るので、**絵柄のレシピとして使い回せる**(CLIの `--config` にも流用可)。

### 出力オプション(メニュー)
- **レイヤー個別PNGも保存**: `_lines.png` / `_flat.png` を並べて出力
- **flow map(UE5用)も保存**: 変位フィールドを `_flow.png` に出力
  → UE5のpost-process materialでひずみをリアルタイム再現する用(README.md参照)

## 4. アーキテクチャ(改造する人向け)

```
studio/
├── app.py             エントリポイント
├── theme.py           ダークテーマQSS
├── main_window.py     全体の組み立て・エクスポート・プリセット
├── canvas.py          ズーム/パン/クリックのキャンバス
├── panels.py          ParamBinder(ドットパス⇔ウィジェット双方向バインド)
├── timeline_widget.py キーフレームUI(engine Timelineへ変換)
└── render_worker.py   QThreadレンダラ+ステージ単位キャッシュ
```

設計のポイント:
- **GUIはエンジン(paintflow/)に一切ロジックを持たない**。パラメータを書いて
  `render_stages()` を呼ぶだけ。エンジンを改造すればGUIにもCLIにも同時に反映される
- **ParamBinder**: `binder.slider("drip.melt", 0, 1, 0.01, 2)` の1行で
  「スライダー⇔params⇔プリセット⇔タイムライン」が全部つながる。
  エンジンに新パラメータを足したら panels.py に1行足すだけでUI化できる
- **ステージキャッシュ** (render_worker.py): 各ステージのparamsのJSONをキーに、
  変わったステージ以降だけ再計算。melt操作時はdripのみ(実測 約100ms vs 全再計算255ms)
- **latest-wins**: スライダー連打時は最新リクエストだけ処理し、古い結果は捨てる

新しいステージをGUIに足す手順:
1. エンジン側: params.pyにdataclass追加 → pipeline処理を書く
2. render_worker.render_stages にキャッシュ段を1ブロック追加
3. panels.py にGroupBoxを追加(binder呼ぶだけ)
4. アニメさせたければ timeline_widget.ANIMATABLE に1行追加

## 5. トラブルシュート

| 症状 | 対処 |
|---|---|
| run_studio.batが一瞬で閉じる | コマンドプロンプトで `python studio_main.py` を実行しエラーを確認。大抵は `pip install -r requirements.txt` 忘れ |
| exeビルドが失敗する | `python -m pip install --upgrade pip pyinstaller` 後に再実行。ウイルス対策ソフトがdistフォルダをブロックする場合は除外設定 |
| プレビューが重い | ツールバーの「プレビュー解像度」を512pxに。書き出しは常にフル解像度でできる |
| 領域が繋がってしまう/分かれすぎる | 「自動彩色→隙間閉じ」を調整(ラフは4〜6)。線が薄い場合は「線画抽出→閾値バイアス」を下げる |
| バケツで思った領域に入らない | レイヤー表示を「領域分け」にすると分割状態が見える |
